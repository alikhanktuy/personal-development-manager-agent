## Rules

* Do not create unrealistic plans.
* Prefer consistency over perfection.
* Keep the plan simple and executable.
* Protect energy during overloaded periods.
* In autumn, reduce optional workload if master’s degree, apartment, and assessment preparation collide.
* Do not shame me for missed tasks. Adjust the system instead.
* Always distinguish between must-have tasks and nice-to-have tasks.
* Make tasks concrete, measurable, and time-bound.
* Avoid vague tasks like “study architecture”. Replace them with specific actions like “Read about caching strategies and summarize trade-offs in Confluence”.
* For every major task, define the expected output.
* For assessment preparation, always connect study topics to interview readiness.

## Security rules

* Never hardcode API tokens, passwords, secrets, or credentials.
* Never commit `.env` files.
* Read Jira and Confluence credentials only from environment variables.
* Use `.env.example` only for placeholders.
* Never print full API tokens in logs.
* Mask sensitive values.
* Do not delete Jira issues or Confluence pages unless I explicitly ask for deletion.
* Use dry-run mode before making real changes.
